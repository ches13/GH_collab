4. Exercise: Collaborating in Git & GitHub - Part 1: Setup

What You'll Do

  • In this exercise, you will set up a GitHub repository to use in practicing commands and concepts for team collaboration.


Instructions

Create remote repository on GitHub

  • We will once again create a remote repository on GitHub, and link it to a local repository on your computer.
  • Log in to your GitHub account at https://github.com/  
  • Once you are logged in, you should see a green New button at the upper left of your screen, to the right of the word Repositories. 
  • Click this button and you will be taken to a form to create a new repository.
  • For the Repository name field, enter GH_collab:

  • Leave the other fields as-is, and click the Create repository button at the bottom of the page.
  • This will create a remote repository hosted by GitHub, and take you to a new page with a few options for what to do next. 
  • Keep this page open. We will return to it in a moment. 

Create local repository on your computer

  • Open a bash terminal, and enter this command to cd into the week1/  folder for this course:

    • cd ~/Desktop/NucampFolder/Python/3-DevOps/week1
  • Next, create a folder with the name github_collab, and cd into it as well: 

    • mkdir github_collab
    cd github_collab

  • Then return to the GitHub page that was shown after you created your remote repository there, and copy the first set of commands. 
  • They should look similar to the below screenshot -- the only difference should be that the URL to the remote repository should point to the one you made:

  • Copy that set of instructions, and paste them into your bash terminal. Be sure to press enter at the last line, which should look like this:


Django app files

  • Download this file and place it in the 3-DevOps/week1/github_collab/ folder.

app.zip
  • From the 3-DevOps/week1/github_collab/ folder in your terminal, enter the following commands to unzip the app.zip file, delete the zipfile, and cd into the unzipped app/ directory:
unzip -q app.zip
rm app.zip

cd app

  • List the contents of the app/ folder, and make sure it appears as follows:
ls

  • These are the files for a basic Django application that have been provided for you as an example. We will discuss in more detail later what Django is and how it works, but for now, these are simply example files for you to practice Git. 
  • Return to the root folder of the local git repository, github_collab/:
cd ..
  • Check the status of the git repository:
git status

 

    • Notice how the app/ folder, which includes the Django app files, is listed as untracked files.  
  • Add, commit and push the Django app files to the remote GitHub repository with the below commands:

git add . 
git commit -m "add django app" 
git push 


  • Refresh your GitHub page for the GH_collab repository (https://www.github.com/<yourusername>/GH_collab) in your browser and confirm the app/ folder and its contents were pushed to the remote repository. 


The .gitignore file

  • A hidden file named .gitignore is used in Git repositories to prevent specified files and folders from being added to the Git repository automatically. 
  • This is useful for various reasons such as security (you would not want to add files that have sensitive data), and to prevent unnecessary files from being added. Some teams prefer not to commit their SQL migrations files, for example. 
  • Let's say that for the example project you have in your app/ folder, the migrations files will be generated in app/helloapp/migrations/.
  • To create a .gitignore file that will "ignore" any future generated migrations files for this project, run this command from a bash terminal in the github_collab/ folder: 
echo app/helloapp/migrations >> .gitignore

  • This creates a file named .gitignore in your current directory and appends the text app/helloapp/migrations in it. 
  • This will cause your local Git repository to ignore that directory.
  • To confirm that the echo command has worked, enter this command:
cat .gitignore

  • This prints the contents of the .gitignore file to the screen. Confirm that you see this output:

  • In preparation for adding a virtual environment folder to this repository, let's add one more line to the .gitignore file by running this CLI command in the github_collab/ folder. This time, it will not create the file because it already exists; instead, it will append the text venv to it:

echo venv >> .gitignore

  • Once again, use the cat command to print the contents of the .gitignore file and confirm that it appears as follows:
cat .gitignore

  • Add, commit, and push the .gitignore file:
git add .
git commit -m "add .gitignore"
git push
  • Confirm in GitHub that the push was made by refreshing the repository page. You should see the .gitignore file there. 


Setup complete

Your setup steps are complete for the Git collaboration exercise. Continue on to Part 2!
